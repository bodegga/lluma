## Two-provider topology (ADR-0002; leak L10 — no vendor sees both edges)

```
client ──TLS──▶  RELAY (Vultr)  ──▶  GATEWAY + co-located ISSUER/BROKER origin (DigitalOcean)
                 sees IP + OHTTP        gateway decapsulates OHTTP, path-allowlists;
                 ciphertext only        broker is sole redeemer; origin bound to loopback
                                              │
                                    broker ──▶ serving hosts (register via ingress; PoW + admission)
```

## Ports / listeners

- Relay: `LLUMA_RELAY_BIND` (public, TLS-fronted) → forwards to the gateway.
- Gateway: `LLUMA_GATEWAY_BIND` (reachable from the relay only) → origin on loopback.
- Broker origin: core `127.0.0.1:8080` (issuer + exec + trial + snapshot), ingress
  `127.0.0.1:8081` (host register/heartbeat/receipt + admin). Loopback-bound; only the gateway
  reaches the core, and hosts reach ingress over the host network path.

## Deploy procedure (summary; scripts in `ops/deploy/`)

1. `~/.lluma-deploy.env` holds `VULTR_API_KEY`, `ZONEEDIT_USER`, `ZONEEDIT_TOKEN`. Key material
   in `~/.lluma-keys-prod`.
2. **VPS-B (DO):** create droplet with the `lluma-deploy-claude` key → install Rust + build
   tools → clone `bodegga/lluma` (main) → `cargo build --release -p lluma-gateway -p lluma-broker`
   → scp key material to `/etc/lluma/keys` (0600) → fill `/etc/lluma/{broker,gateway}.env` →
   install `ops/deploy/systemd/*.service` → start broker then gateway → firewall (gateway port
   from relay only; origin loopback).
3. **VPS-A (Vultr):** create instance with the deploy key → build `lluma-relay` → set
   `LLUMA_RELAY_GATEWAY` to VPS-B → start relay → TLS-front it.
4. **DNS (ZoneEdit):** `relay.lluma.bodegga.net` → VPS-A, `gw.lluma.bodegga.net` → VPS-B; issue
   Let's Encrypt certs.
5. Register a serving host against the broker ingress (PoW + heartbeats to admission).
6. End-to-end smoke test with `lluma-client`; verify the relay sees only IP+ciphertext and the
   origin never logs prompt/response plaintext.
7. **Site:** `doctl apps create-deployment c8effc7b-…` after pushing site changes to `main`.
